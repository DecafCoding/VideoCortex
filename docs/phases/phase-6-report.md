## Phase Description

This phase turns a project's `index.html` into a **complete, synthesized cross-video report** — not a bare list of links, but multi-section semantic HTML that reads like a briefing on everything the project's videos cover, cites the individual per-video concept pages inline, and ends with a **Sources** section enumerating every video. The report is (re)generated by an LLM over the **compact per-video summaries** produced in Phase 5 (`SummaryTitle` / `SummaryDescription` / `SummaryBodyMd`) plus the project's `AIInstructions`, wrapped deterministically in the OKF **root-index template**, and written to `<Project folder>/index.html`.

Generation is driven by a new **debounced `ReportWorker`** (a polling `BackgroundService`) that fires whenever a video reaches `Summarized`, coalescing rapid successive additions into a single expensive LLM call, and by an explicit **"Rebuild report"** button on the project detail page. Removing a video deletes its concept page and re-marks the report dirty so the next pass folds the removal in. Two "open on disk" links are wired: **"Open library ↗"** (to the root `index.html`) and per-video **"open page"** (to each concept file).

This closes the MVP pipeline: `Added → Transcribed → Summarized → Published`. A video reaches `Published` once its content has been folded into a successfully-regenerated report.

This is a greenfield repository. The reference implementation for every pattern is **SkipWatch** at `C:\Repos\SkipWatch\SkipWatch` — its wiki compiler + debounced wiki worker are the direct analogue of this phase's report synthesizer + report worker. Read those files to copy conventions, then write clean, minimal equivalents. Do **not** add a project reference to SkipWatch and do **not** copy its job-queue / override / citation-store machinery — Video Cortex deliberately uses a simpler per-project dirty-timestamp trigger.

## User Stories

As the single local user
I want the project's `index.html` to be a complete, multi-section report across all its videos — updated automatically as I add videos and on demand via a button
So that opening the library in my browser gives me synthesized understanding of the whole topic, with citations back to each video's page, not just a pile of links.

As the single local user
I want "Open library ↗" and per-video "open page" links, and a "remove video" action that also cleans up the report
So that the on-disk artifact stays consistent with what I see in the app and is usable independent of it.

As the operator
I want report regeneration debounced and resilient
So that pasting five videos in a row triggers one LLM call, not five, and a synthesis failure never destroys the last good `index.html`.

## Problem Statement

After Phase 5 each summarized video has its own OKF concept page, but the project's root `index.html` is still the empty report shell seeded at project creation (Phase 2). There is no cross-video synthesis, nothing that folds new summaries into a coherent whole, no "rebuild" affordance, no way to open the on-disk artifact from the app, and removing a video leaves an orphaned concept page and a stale report.

## Solution Statement

Add `IReportSynthesizer.SynthesizeAsync(projectName, aiInstructions, videoSummaries[])` (a Phase-5 `OpenAiClient`-backed call using `response_format = json_schema`, output schema `{ library_description, report_html }`) that returns semantic inner-body HTML organized into thematic `<h2>` sections with inline relative citations and a closing Sources `<section>`. Extend the Phase-2/5 `OkfLibraryStore` with `WriteReportAsync` (wraps the LLM body in the deterministic OKF root-index template and atomically writes `index.html`) and `DeleteConceptPageAsync`. Drive regeneration with a **per-project "report dirty since" timestamp** + a debounced `ReportWorker` (mirroring SkipWatch's `WikiWorker`), setting `Project.Status = Updating` during and `ReportUpdatedAt` after, and flipping each folded-in `Video.Status` to `Published`. Wire the "Rebuild report", "Open library ↗", per-video "open page", and "remove video" UI. Guard the previous good `index.html` on synthesis failure. Cover the synthesizer (mocked `HttpMessageHandler`), the writer (temp-dir HTML assertions), and the debounce logic with tests that touch no network.

### Patterns to Follow

**Naming**: PascalCase types/methods, `I`-prefixed interfaces, `sealed` service classes, records for contexts/DTOs/results, one type-group per file. Namespaces `VideoCortex` (host) / `VideoCortex.Core` (domain). New folders: `VideoCortex.Core\Services\Report\`, `VideoCortex\Workers\`.

**Synthesizer** (mirror `OpenAiWikiCompiler`): one `IOptionsMonitor<LlmSettings>` read per call; build system+user messages with a `StringBuilder`; POST via the `OpenAiClientCache`-supplied `HttpClient` to `/v1/chat/completions`; on non-2xx throw with the endpoint's error body; parse `Choices[0].Message.Content` (a JSON string) into `ReportSynthesisResult`; wrap parse failures in a typed `ReportSynthesisException`.

**Runner** (mirror `WikiIngestRunner` but single-artifact): load project + its videos; **gate** — only summaries with non-empty `SummaryBodyMd` are eligible; if none eligible, no-op without an LLM call; set `Project.Status = Updating`; synthesize; on success `WriteReportAsync`, flip eligible videos `Summarized → Published`, set `Project.Status = Idle`, `ReportUpdatedAt = now`, `ReportDirtySince = null`; on failure keep prior file, backoff `60s × 2^(n-1)` capped 1h, park after `MaxRetryAttempts` with `Project.Status = Error`.

**Worker** (mirror `WikiWorker`): `TickOnceAsync` selects the project with the oldest non-null `ReportDirtySince` that is eligible (past backoff); debounce `CoalesceDebounceSeconds` from `ReportDirtySince`; idle-sleep `IdlePollSeconds` when nothing to do; swallow `OperationCanceledException` on shutdown; every DB access via a fresh DI scope from `IServiceScopeFactory`.

**Disk safety** (reuse `OkfLibraryStore` conventions): writes confined to `RootPath\<slug>\`; validated filenames; atomic temp-then-move; UTF-8 no BOM; never touch sibling libraries.

**Warnings-as-errors**: everything compiles clean under `TreatWarningsAsErrors=true`; no unused usings, no nullable warnings.

### Unit Tests
- Synthesizer builds a correct `json_schema` request and parses `ReportSynthesisResult`; surfaces endpoint errors; wraps malformed output in `ReportSynthesisException` (mocked `HttpMessageHandler`, no network).

- Rebuild on a parked project → clears backoff and retries.
- Scalar placeholders containing `&`/`<`/quotes → escaped, not raw, in the written index.

## NOTES

- **Base branch is `main`** (repo default + the user's PR-based-on-main workflow). Phase branch: `phase-6-report` off `main`. **Do not** add Claude as commit author or `Co-Authored-By` — the repo convention is that all commits are authored solely by the user.

- **Trigger design — chosen: per-project "report dirty since" timestamp + debounce, NOT a job-queue table.** SkipWatch uses a full `ProjectWikiJob` queue (queued/running/done/parked rows, per-job retry, add/remove actions) because it debounces *per-page* multi-op plans across add AND remove batches. Video Cortex regenerates a **single** artifact from the **whole** current summary set each pass — there is nothing to batch at the row level, so a queue table is pure overhead. A single nullable `Project.ReportDirtySince` column (set by summary completion / rebuild / remove, cleared on success) is the entire trigger; the worker debounces from it exactly as `WikiWorker` debounces from `EnqueuedAt`. This is simpler, has no orphaned-job cleanup, and keeps the worker SELECT to one indexed column. The one cost is backoff/park bookkeeping needs a home — handled by two small `Project` columns (`ReportRetryCount`, `ReportNextAttemptAt`) rather than a job row (see Task 5 GOTCHA; fold into the Task 4 migration or add a second migration — the implementer decides and records which).

- **LLM returns inner body HTML; the OKF skeleton + `okf-meta` are a deterministic wrapper.** This is the single most important structural rule: the model must NOT emit `<!doctype>`/`<html>`/`<head>`/`<body>`/`okf-meta`/`<h1>` (the wrapper owns those). Constraining the model to inner-body HTML is what keeps every generated library byte-consistent with the hand-built ones and keeps conformance deterministic regardless of model quality. `report_html` is injected raw (it is meant to be HTML); only scalar placeholders (`title`, `library_description`) are escaped per context.

- **Keep-prior-file-on-failure.** A synthesis or write failure must never destroy the last good `index.html`. The runner never calls `WriteReportAsync` on the failure path, and `WriteReportAsync` itself is atomic (temp-then-move), so a crash mid-write leaves the prior file intact. This is asserted with a byte-equality test (Task 9c).

- **"Open library" via `file:///` href (chosen) vs. an app endpoint (alternative).** For a single-user localhost app the simplest reliable "open the on-disk artifact" is a `file:///` href built with `new Uri(absolutePath).AbsoluteUri` (handles the spaces in `Documents\SecondBrain\<Project Name>` and the drive letter correctly). An app endpoint that `Process.Start`s Explorer to reveal the file is the alternative but adds a server code path and OS coupling with no MVP benefit — deferred. Some browsers block `file://` navigation from an `http://localhost` page; if that surfaces in manual testing, fall back to displaying the absolute path with a copy button. Document whichever ships.

- **Context size is fine for MVP; chunking is v2.** Input is the compact per-video summaries (`SummaryTitle`/`SummaryDescription`/`SummaryBodyMd`), never transcripts, so many videos fit one context window (PRD §6.5, Risks). Very large projects (100+ videos) overflowing a single window are a v2 map-reduce concern — flag it, do not build it here.

- **Enum contract.** `VideoStatus` order (`Added, Transcribed, Summarized, Published, NoTranscript, Error`) and `ProjectStatus` (`Idle, Updating, Error`) are persisted ints from Phase 1 — this phase only *reads/writes* `Summarized`/`Published` and `Idle`/`Updating`/`Error`; do not reorder.

- **No SkipWatch dependency.** SkipWatch is a read-only pattern reference; Video Cortex shares no code or project reference with it. The wiki compiler/worker/runner informed the synthesizer/worker/runner shapes only.

- **Remote/PR precondition.** As in prior phases, no `origin` remote or authenticated `gh` may exist at execution time. The final task documents that push + PR require a remote and `gh auth`; absent that, the phase ends at a local commit.
