# Video Cortex

**Turn YouTube videos into a compounding, per-project knowledge library.**

Video Cortex is a single-user, locally installed **.NET 10 Blazor Server** app. You create a **Project** for a research topic, paste in YouTube videos, and the app fetches each transcript, summarizes it with an LLM, and maintains a synthesized **report** across everything the project's videos cover.

The durable output is written to disk as **OKF-HTML** (Open Knowledge Format, HTML profile) under `Documents\SecondBrain\<Project Name>\` — each project is a standalone library (one `theme.css`, a root `index.html` report, and one concept page per video) that opens in any browser and outlives the app itself.

Video Cortex is a deliberately simplified, standalone descendant of **SkipWatch**: it reuses the pipeline patterns but drops channel-following, discovery, triage, quota management, and the MCP server — leaving just **paste a video → get a knowledge report**.

## What it does

- **Projects** — create a research topic; each one becomes a standalone OKF-HTML library folder on disk.
- **Manual add** — paste a YouTube URL (any common shape) or a bare video ID; the video is queued for processing.
- **Transcribe** — fetch the transcript and metadata (Apify). *(pipeline stage)*
- **Summarize** — an LLM writes a per-video concept page. *(pipeline stage)*
- **Report** — an LLM synthesizes a cross-video report as the project's `index.html`, growing as videos are added. *(pipeline stage)*

Pipeline, end to end:

```
Paste URL → Added → Transcribed → Summarized → Published
                     (Apify)      (LLM → page)  (LLM → report)
```

## Tech stack

- **.NET 10** — Blazor Server (Interactive Server components)
- **EF Core 10 + SQLite** — local state store at `~/.videocortex/app.db`
- **Apify** — YouTube transcript + metadata (no YouTube Data API)
- **OpenAI-compatible LLM** — hosted OpenAI or a local Ollama `/v1` endpoint
- **OKF-HTML** — Open Knowledge Format (HTML profile), the on-disk output format
- **xUnit + FluentAssertions** — tests, in-memory SQLite fixtures

## Getting started

```bash
# Build, test, run
dotnet build VideoCortex.slnx
dotnet test  VideoCortex.slnx
dotnet run   --project VideoCortex/VideoCortex.csproj
```

The app runs at the URL shown in the console (see `VideoCortex/Properties/launchSettings.json`). EF Core migrations are applied automatically at startup, creating `~/.videocortex/app.db` on first run.

### Secrets (development)

Non-secret settings live in `VideoCortex/appsettings.json`; **secrets are blank in source** and supplied via user-secrets:

```bash
cd VideoCortex
dotnet user-secrets set "Apify:Token" "apify_api_..."
dotnet user-secrets set "Llm:ApiKey"  "sk-..."
```

At runtime, settings can also be written to a hot-reloaded overlay at `~/.videocortex/appsettings.Local.json`.

## Configuration

All knobs are in `VideoCortex/appsettings.json` (bound to strongly-typed records):

| Section | Key settings |
|---|---|
| `Apify` | `Token` (secret), `RunTimeoutSeconds`, `PreferredLanguage`, `PreferAutoGenerated` |
| `Llm` | `Model` (default `gpt-4o-mini`), `BaseUrl` (blank → hosted OpenAI; set to Ollama `/v1` for local), `ApiKey` (secret), `RequestTimeoutSeconds` |
| `Library` | `RootPath` (default `Documents\SecondBrain`) |
| `TranscriptWorker` / `Summary` / `Report` | `IdlePollSeconds`, `MaxRetryAttempts`, and (Report) `CoalesceDebounceSeconds` |

## Layout

```
VideoCortex/                  Blazor Server host — UI, workers, DI
  Features/Projects/          Project list/detail, add-video form, video table
  wwwroot/okf/                Bundled OKF templates (theme.css, concept.html, index.html)
VideoCortex.Core/             Domain library (UI-free)
  Entities/                   Project, Video, enums
  Db/                         VideoCortexDbContext (+ migrations)
  Services/Library/           OKF-HTML disk writer
  Services/Triage/            YouTube input parser, video commands/queries
  Services/Config/            Runtime paths + settings records
VideoCortex.Tests/            xUnit + FluentAssertions
docs/                         PRD, phase plans, progress tracker
```

## Data & output locations

- **State DB:** `~/.videocortex/app.db` (SQLite)
- **Config overlay:** `~/.videocortex/appsettings.Local.json`
- **Libraries (durable artifact):** `Documents\SecondBrain\<Project Name>\`
